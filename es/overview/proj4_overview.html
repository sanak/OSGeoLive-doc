Proj.4¶

Proj4 es una biblioteca que proporciona métodos para transformar entre sistemas de referencia de coordenadas diferentes. Un sistema de referencia de coordenadas geográficas permite que todos los puntos de la tierra puedan ser descritos como un conjuntos de coordenadas (como latitud, longitud y altitud). Diversos sistemas se utilizan para representar la tierra tridimensional en un plano o mapa bidimensional.

Proj4 se usa en MapServer, GRASS GIS, PostGIS, Mapnik, Thuban, OGDI, TopoCad, OGRCoordinateTransformation y varios otros.

Proj4 ha sido traducido a partir de C a otros lenguajes <https://trac.osgeo.org/metacrs/wiki>. Estos subproyectos incluyen: Estas herramientas incluyen: * Proj4js (JavaScript) - used in OpenLayers. * CS-Map (C++) - used in MapGuide. * GeoTIFF/libgeotiff (C++). * Proj4J (Java). * SpatialReference.org - sitio web para recuperar las definiciones del sistema de coordenadas (obsoleto).

Características Principales¶

  • Proporciona el punto de transformación de un sistema de referencia de coordenadas a otro.
  • Incluye transformación entre datums (datos de referencia).
  • Gran cantidad de clases de proyección compatibles.

Estándares Implementados¶

Proj4 utiliza las definiciones del sistema de coordenadas dadas por el European Petroleum Survey Group (EPSG) <https://www.epsg.org/>. Bien conocido formato de texto (WKT).
