Inicio Rápido de MapSlicer¶

MapSlicer proporciona una forma sencilla de crear un conjunto de teselas de cualquier imagen de mapa georeferenciado. Las teselas pueden entonces presentarse como superposición de mapa georeferenciado en un servidor web. Se adecúa a las especificaciones de Open Source Geospatial Foundation’s (OSGeo’s) Tile Map Service (TMS)

  • Opciones de entrada: Cuañquier archivo de imagen georreferenciada, como TIFF, JPEG de ArcGIS, QGIS.
  • Opciones de Salida: Un conjunto de directorios con teselas imágenes que se pueden copiar a un servidor web para su presentación como una superposición georeferenciada.
  • Fuerzas: MapSlicer es un software de código abierto, simple, libre, con un asistente fácil, que se describe a continuación, para guiar a través de las opciones.

MapSlicer aprovecha el hecho de que Google Maps, Microsoft Bing, Yahoo Maps, y otros proveedores de mapas en línea, incluido <https://www.openstreetmap.org/>`__ usan el mismo perfil de proyección y mosaico y, por lo tanto, los mosaicos son compatibles. MapSlicer prepara teselas utilizando esta especificación como una forma fácil y rápida de preparar un mapa georreferenciado en una página web de mashup.

Iniciar MapSlicer¶

  1. Iniciar MapSlicer en el menú Geoespacial, bajo el submenú Herramientas Espaciales.

  2. Seleccionar Tile Profile. Elegir Image Based Tiles (raster) para publicación estándar en web. Click Continue.

  3. Elegir el Source Data Files. Examine para seleccionar la imagen ráster que desea como tesela. También es posible seleccionar un color NODATA que aparecerá como transparente en la imagen resultante. Haga click en*Continuar*.

  4. Especificar la Sistema de Referencia Espacial / Sistema de Coordenadas de la imagen. Especificar el número de identificación de la base de datos de la EPSG. Haga click en Continuar.

  5. Especifique detalles sobre el Tile Pyramid. La configuración predeterminada para los niveles de zoom y el formato de archivo suele ser la mejor. Haga click en Continuar.Detalles

  6. Especificar detalles sobre la Carpeta de destino y Direcciones / URLs para el conjunto de mosaicos. Si no los conoce, podrá agregarlos en los archivos predetermionados de googlemaps.html y openlayers.html después de generar el mosaico. Haga click en Continuar.

  7. Marque los Visores que deberían generarse. De forma predeterminada, se genera un archivo openlayers.html. Haga click en Continuar.

  8. Especifique los Detalles para generar Visores, como el título, aviso de copyright y las claves de API. Si no los conoce, se pueden agregar en los archivos predeterminados de googlemaps.html / openlayers.html tras generación del mosaico. Haga click en Continuar.

  9. Haga click en Render para comenzar a representar la imagen.

  10. Cuando termine, MapSlicer proporciona un vínculo al conjunto de mosaicos terminado.

  1. Abra los archivos openlayers.html en un explorador web para ver el conjunto de mosaicos como una superposición en el mapa en línea vivo. Para presentar esto en la web, copiar el conjunto de mosaicos entero y todos sus subdirectorios en un servidor web y editar los archivos googlemaps.html o openlayers.html según sea necesario.

Utilizar gdal2tiles para crear una base de datos de MBTiles¶

MapSlicer es esencialmente una interfaz GUI al script de gdal2tiles.py. Una versión actualizada de gdal2tiles.py también está presente en el disco Live. Los árboles de mosaicos resultantes creados por este software se pueden usar con el programa mb-util para crear una base de datos mbtiles, para usar con software de teléfonos inteligentes como` Geopaparazzi <https://www.geopaparazzi.org/>`_.

Para crear una base de datos MBTiles SQLite del árbol de mosaico recién creado, abra un terminal, entonces

cp data/raster/bluemarble.tif .
mapslicer    # (select /home/user/bluemarble.tif for input, JPEG as format)
mb-util --scheme=tms --image_format=jpg bluemarble bluemarble.mbtiles
qgis bluemarble.mbtiles

Para mayor información sobre la ejecución de estas herramientas, ver gdal2tiles.py –help y mb-util –help
