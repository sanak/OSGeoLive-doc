MapSlicer gyorstalpaló¶

A MapSlicer egyszerű lehetőséget biztosít csempék létrehozására georeferált raszterekből. A csempéket ezután mint egy georeferált térkép réteget jeleníthetjük meg egy web szerveren. Az Open Source Geospatial Foundation (OSGeo) Tile Map Service (TMS) specifikációját valósítja meg.

  • Input beállítások: Tetszőleges georeferált kép fájl pl. TIFF, JPEG ArcGIS, QGIS szoftverekből.
  • Output: A könyvtárak egy csoportja a csempe képekkel, melyeket egy web szerverre másolhatunk egy georeferált rétegként történő megjelenítéshez.
  • Erősségek: MapSlicer egyszerű, szabad, nyílt forráskódú program egy könnyen használható varázslóval, melyet alább írunk le, hogy végigvezessük a beállításokon.

A MapSlicer kihasználja azt a tényt, hogy a Google Maps <http://maps.google.com/> _, a Microsoft Bing <http://maps.bing.com> _, a Yahoo Maps és más online térképszolgáltatók, beleértve az OpenStreetMap -et, ugyanazt a vetületet és csempézési profilt használják, és a így a csempék kompatibilisek. A MapSlicer ezen a specifikációalapján készíti a csempéket, mint egy könnyű és gyors módszer a georeferált térkép készítésére egy mashup honlapon.

MapSlicer indítás¶

  1. Indítsa el a MapSlicer-t a GeoSpatial menüből, a Spatial Tools almenü alatt.

  2. Válassza a Tile Profile-t. Majd az Image Based Tiles (raster) a szabványos webes publikáláshoz. Kattintson a Continue-ra.

  3. Válassza a Source Data Files-t. Tallózzon a csempékre darabolandó raszterképhez. Lehetőség van a NODATA szín kiválasztására is, ez átlátszóként jelenik meg az eredmény képen. Kattintson a Continue-ra.

  4. Adja meg a kép vetületi rendszerét Spatial Reference System / Coordinate System. Adja meg az azonosító számot az EPSG adatbázisból. Kattintson a Continue-ra

  5. Adja meg a Tile Pyramid részletei. Az alapértelmezett beállítások a nagyítási szintekre és a fájl formátumra gyakran a legjobbak. Kattintson a Continue-ra.

  6. Adja meg a cél könyvtárat és címet (Destination folder and Addresses / URLs) a csempehalmazra. Ha nem ismeri ezeket, hozzáadhatók a csempegenerálás utána az alapértelmezett googlemaps.html és openlayers.html fájlokhoz. Kattintson a Continue-ra.

  7. Jelölje ki a generálandó nézeteket. Alapértelmezés szerint egy openlayers.html fájl jön létre. Kattintson a Continue-ra.

  8. Adja meg a Details for generating the Viewers (részletek a nézegető generálásához), mint a cím, szerzői jog figyelmeztetés és API kulcsok. Ha nem tudja ezeket, utólag hozzáadhatók az alapértelmezett googlemaps.html / openlayers.html fájlokhoz a csempe generálás után. Kattintson a Continue gombra.

  9. Kattintson a Render gombra a kezdéshez.

  10. Amikor befejezte, a MapSlicer megadja a linket az elkészített csempekészletre.

  1. Nyissa meg egy web böngészőben openlayers.html fájlt a csempekészlet megtekintéséhez egy elő on-line térképen. A weben történő megjelenítéshez másolja a teljes csempekészletet és minden alkönyvtárat egy web szerverre és módosítsa a googlemaps.html vagy openlayers.html fájlokat szükség szerint.

gdal2tiles használta egy MBTiles adatbázis létrehozására¶

A MapSlicer alapvetően egy grafikus felület a gdal2tiles.py szkripthez. A gdal2tiles.py egy aktualizált változata is rajta van a Live lemezen. Ezzel a szoftverrel létrehozott eredmény csempe fák használhatók  mb-util programmal, hogy mbcsempe adatbázist hozzon létre az okostelefon programokhoz mint a Geopaparazzi.

Az újonnan létrehozott csempe fából egy MBTiles SQLite adatbázis létrehozásához nyisson meg egy terminált, majd

cp data/raster/bluemarble.tif .
mapslicer    # (select /home/user/bluemarble.tif for input, JPEG as format)
mb-util --scheme=tms --image_format=jpg bluemarble bluemarble.mbtiles
qgis bluemarble.mbtiles

Lásd gdal2tiles.py –help és mb-util –help további információkért ezen eszközök futtatásáról.
